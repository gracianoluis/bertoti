3 - Trade Offs 

   Risco vs. Retorno (Investimentos)
Este é o trade-off mais conhecido no mundo das finanças.
A Escolha: Para buscar lucros maiores, você deve aceitar uma chance maior de perder dinheiro.
Como funciona: Se você quer segurança total, investe em algo como o Tesouro Direto ou Poupança, mas o rendimento será baixo. Se deseja dobrar seu capital rapidamente (retorno alto), precisará investir em ativos voláteis como ações ou criptomoedas, onde o risco de desvalorização é constante.
   Qualidade vs. Preço (Consumo)
Este trade-off guia quase todas as decisões de compra no dia a dia.
A Escolha: Equilibrar o nível de sofisticação/durabilidade de um produto com o valor que você está disposto a pagar.
Como funciona: Uma empresa pode fabricar um smartphone com materiais premium, câmeras de última geração e processadores ultra-rápidos (Alta Qualidade), mas o preço final será muito elevado. Para vender um modelo barato (Baixo Preço), ela terá que abrir mão de certas funcionalidades ou usar materiais mais simples.
  Trabalho vs. Lazer (Gestão do Tempo)
Este é um trade-off de alocação de recursos escassos — neste caso, o tempo.
A Escolha: Decidir quanto tempo dedicar à produção de renda versus quanto tempo dedicar ao descanso e vida pessoal.
Como funciona: Se uma pessoa decide fazer horas extras ou pegar um segundo emprego, ela aumenta sua renda e segurança financeira, mas perde horas de sono, tempo com a família ou hobbies. Se ela prioriza o lazer, ganha bem-estar imediato, mas abre mão do ganho financeiro que aquelas horas poderiam gerar.

4 – Tecnicamente software é intangível pois o mesmo não pode ser tocado como um produto físico, porém aí que está o problema, justamente por não ser algo físico é mais fácil de esconder complexidade e uma deterioração lógica (obsolescência).

5 - Programar é o ato de construir o código, Engenharia é um aglomerado de conceitos e práticas que são usadas para uma maior vida útil do software, e que ele funcione para todos os usuários, escalabilidade, e que as decisões de custo benefício sejam bem estruturados. engenharia de software é sobre a sustentabilidade do software ao longo dos anos. 

6 - Escalabilidade é a capacidade do sistema de continuar funcionando bem enquanto o número de usuários ou requisições cresce, Disponibilidade define quanto tempo o sistema fica "no ar". É medido pelos "noves", segurabilidade o nível de proteção que o sistema tem contra ataques para manter os dados seguros, testabilidade o quão fácil e rápido é criar testes para garantir que o código realmente faz o que deveria fazer, e manutenibilidade/Agilidade a facilidade de fazer mudanças, corrigir bugs ou adicionar novas funções sem quebrar o que já existe.

7 - Segurança vs. Usabilidade: Se você colocar 10 etapas de autenticação (biometria, token, senha forte), o sistema fica super seguro, mas o usuário vai odiar porque é difícil de acessar. O trade-off aqui é: quanto mais segurança, geralmente menos conveniência para o usuário.
  Tempo de Desenvolvimento (Time-to-market) vs. Qualidade de Código: Às vezes uma empresa precisa lançar um produto rápido para ganhar o mercado (prazo curto). O trade-off é que, para entregar rápido, os desenvolvedores podem acumular "dívida técnica", escrevendo um código menos elegante que será mais difícil de manter depois.
  Consumo de Memória vs. Velocidade de Processamento: Você pode criar um algoritmo que "decora" (faz cache) de todos os resultados para ser instantâneo na próxima vez que alguém perguntar. O benefício é a velocidade, mas o trade-off é que seu programa vai gastar muito mais memória do computador para guardar essas informações.
